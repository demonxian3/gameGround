实现“一屏全显”布局

1. 锁定视口高度 (Viewport Height Lock)

请将整个页面（Container）的高度设置为 100vh，并添加 overflow: hidden; 以确保页面绝对不会出现全局滚动条。

确保所有子元素都在这一屏高度内进行伸缩适配，而不是将容器撑开。

2. 核心内容区（Game Board）弹性优先

设置中心的游戏画布区域（Game Play Area）为 Flex-grow: 1，使其成为页面中唯一可以自动压缩或拉伸的部分。

同时，请设置 min-height: 0; 给游戏容器，防止它被内部的 Canvas 或 Grid 强制撑大。

3. 顶部与侧边栏的“高度脱水”

顶部栏（Header）： 大幅压缩高度。将“俄罗斯方块”大标题缩小，并与右侧的房间按钮、用户头像强制排在一行，减少垂直空间的占用。

侧边栏（Sidebar）： 减小左侧和右侧面板的内边距（Padding）和间距（Gap）。如果有必要，请缩小“玩法说明”和“快捷操作”的字号，确保它们不会把整体高度向下顶。

4. 自动缩放机制 (Scaling)

如果内部的游戏网格（Tetris Grid）仍然过大，请使用 CSS 的 transform: scale() 或 aspect-ratio 属性，让游戏网格根据父容器剩余高度自动缩小，以确保底部的 UI 元素能完全显露在视口内。

“动态自适应”布局修正指令

1. 容器高度锁定 (Container Lock)

请确保最外层容器（Main Wrapper）的高度严格等于 100vh，并保持 overflow: hidden;，禁止出现任何页面滚动条。

2. 核心区域“自动撑满” (Auto-Expand)

移除之前对中间游戏内容容器（Game Container）设置的所有固定高度（如 height: 500px 或 max-height）。

将游戏内容容器设置为 flex-grow: 1;，使其自动吸收页面底部剩余的所有空白区域，直到触碰视口底边缘。

3. 内部元素比例适配 (Responsive Scaling)

确保游戏容器内部的 Canvas 或 Grid 网格使用 height: 100%; 和 object-fit: contain;，这样它们会随容器的撑开而自动变大，但绝不会超出容器边界。

如果底部仍有空隙，请检查并移除外层容器多余的 padding-bottom 或 margin-bottom。

4. 侧边栏对齐 (Sidebar Sync)

让左右两侧的面板（玩法说明、快捷键等）也采用 display: flex; flex-direction: column;，并将中间的装饰性间距设为 flex-grow: 1;，使两侧面板的高度在视觉上与中间的游戏区保持等高齐平。
