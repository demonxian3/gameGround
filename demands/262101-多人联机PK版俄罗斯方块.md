1. 系统架构与通信
   技术栈： 使用 WebSocket (如 Socket.io) 实现实时双向通信。

房间系统： \* 用户进入页面后自动生成/加入一个唯一的房间号 (Room ID)。

房主 (Host)： 队列最左侧的第一位用户默认为房主。

邀请机制： 队列最右侧有一个“+”号按钮（与头像风格一致），点击后生成当前房间的 URL 链接并自动复制到剪贴板。

2. 玩家队列与 UI (顶部 Bar)
   头像展示： 顶部以横向队列形式展示所有房间内玩家的 Avatar。

用户信息： \* 新用户进入后默认生成随机名称。

支持用户点击自己的头像或名称进行实时修改/编辑。

状态同步： 所有用户的进入、退出、更名操作需通过 WebSocket 广播给房间内所有人。

3. PK 核心机制 (对战逻辑)
   分屏对战： 核心画布区域（俄罗斯方块面板）由单人模式拆分为左右两个分屏。

数值面板：当前状态，

左侧： 当前擂主 (Current Winner/Defender)。

右侧： 当前挑战者 (Challenger)。

排队规则： 队列的前两名自动上场 PK，其余玩家在队列中等待并实时观战。

胜负判定： \* 时间限制： 单场比赛最大时长为 5分钟，可配置

判定条件： 1. 对方提前 GameOver 则己方胜；2. 5分钟倒计时结束，得分高者胜。

流转逻辑： \* 胜利者： 保留在左侧席位，等待队列中的下一位挑战者。

失败者： 自动移动到队列的最末尾，重新进入排队序列。

4. 实时同步数据 (Data Packets)
   游戏数据： 每一帧或每一次方块移动/消除，需同步当前方块矩阵、得分、消除行数、下一个方块 (Next) 给对方及所有观战者。

操作指令： 确保延迟补偿，保证双方画面同步。

概述：
“请基于 WebSocket 改造现有的俄罗斯方块代码，实现多人联机功能。要求：顶部显示玩家头像队列，支持随机更名和链接邀请；实现分屏 PK 机制，队列前两名自动对战，限时5分钟，采用‘胜者守擂、败者排队’的规则。请给出前端 UI 调整建议以及后端 Socket 处理逻辑的关键代码实现。”

UI 布局与数据展示补充说明

1. 局部容器重构 (蓝圈部分)
   分屏化处理： 仅对游戏的核心对战区域（即原单机版方块降落的画布区）进行横向拆分。

左右布局： 画布区拆分为左、右两个独立的容器。左侧固定为擂主（Current Defender），右侧固定为挑战者（Current Challenger）。

观战模式： 对于队列中等待的其他玩家，这两个容器将实时同步对战双方的画面。

2. 全局侧边栏保持 (红圈部分)
   左侧导航栏： 保持现有的项目标题 Arcadia Grid、游戏列表及“玩法说明”不变。

右侧状态栏： 保持布局结构。但需将“当前状态”中的信息逻辑进行调整，主要展示房间全局信息。

3. 实时对战数据展示
   倒计时显示： 在右侧“当前状态”栏（或屏幕正上方显著位置）新增一个 “剩余时间” 计数器，从 05:00 开始实时倒计时。

得分浮层： \* 取消原本集中在右侧面板的单一分数值展示。

玩家得分需直接显示在各自的俄罗斯方块画布边缘（例如：在各自画布的顶部或侧边显示动态数值），以便于观众和选手直观对比。
