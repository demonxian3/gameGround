请按照以下详细需求对当前页面进行代码实现和优化：

1. 全局按键映射扩展 (Input Mapping)
   需求： 所有的上下左右操控逻辑，除了原有的方向键外，必须同步支持 WASD。

技术细节： \* W 对应 ArrowUp (旋转/辅助)

A 对应 ArrowLeft (左移)

S 对应 ArrowDown (下落)

D 对应 ArrowRight (右移)

请确保在监听 keydown 事件时，这两种键位的 event.code 或 event.key 触发相同的业务函数。

2. 新增模式切换控件 (UI Component)
   位置： 在界面右上角红圈位置（“复制邀请链接”按钮左侧）增加一个切换控件。

样式： 参考 Ant Design 的 Segmented（分段控制器） 组件样式。

选项：

单人模式 (Single Player)

联机模式 (Multiplayer)

3. 状态持久化 (Persistence)
   需求： 页面刷新后应保持用户之前的选择，避免重复操作。

技术细节： \* 使用 localStorage 记录当前的游戏索引 (Game ID) 以及运行模式 (Single/Multi)。

页面 onMount (初始化) 时，优先从缓存读取状态并恢复 UI 和逻辑。

4. 模式切换逻辑与 WebSocket 自动重连
   场景描述： 当用户从“单人模式”手动切换到“联机模式”时。

判断逻辑：

检查当前浏览器 URL 参数（例如 ?room=5137J7）。

如果存在有效的 room 参数，自动触发 WebSocket 连接流程，加入对应的房间。

如果不存在参数，则默认进入联机大厅或等待匹配状态。

如果 Codex 没生成 UI： 提醒它“请使用现有的 CSS 变量和 Flex 布局来确保新控件与右上角按钮对齐”。

关于持久化： 记得提醒它在 onChange 事件发生时立即更新 localStorage

调换一下模式和复制链接的位置，保持无论是单机/联机，模式切换的控件位置不变
